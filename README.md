# Task_signage Ver.1

Google Tasks のタスクと Google カレンダーの当日予定をサイネージモニタに常時表示する、
Google Apps Script (GAS) 製のWebアプリです。タッチ／マウス操作によるタスク操作にも対応しています。

## 構成ファイル

| ファイル | 役割 |
|---|---|
| `Code.gs` | サーバー側。Tasks / People / Chat / Calendar 各APIとの通信、設定の保存 |
| `index.html` | 画面側。3パネルレイアウト、時計、ボトムシートUI、自動更新 |

## 主な機能

### タスク表示（ToDoパネル）
- 自分の**全タスクリストを統合**して取得
- 「今日」「今週中（明日〜7日後）」「期限切れ」の3列表示。各列見出しにタスク数を `(n)` 表示
- 期限なしタスクは「今日」列に表示
- Google Chat スペース／Google Docs で自分にアサインされたタスクも表示（`タスク名（スペース名）`形式）
- 当日完了したタスクは「今日」列の末尾に取り消し線で表示
- 期限切れは超過日数を赤字で表示
- 並び順は「期限順／Google Tasksの順／タイトル順」から選択可

### タスク操作（タッチ／マウス）
- **○タップで完了**（即座に取り消し線表示、5秒間のUndoトースト付き）。✓再タップで完了取り消し
- **⋮メニュー（ボトムシート）**
  - 明日に延期（ワンタップ）
  - 期限を変更（日付ピッカー）
  - 依頼メールを送る（宛先のインクリメンタル検索、文面3種: 標準／丁寧／簡潔、依頼期限指定、アドレス直接入力も可）
  - スマホで編集（そのタスクを Google Tasks で開くQRコード）
  - 詳細を表示
  - 削除（確認付き。**アサイン由来のタスクでは非表示**＝スペース側の全員分を誤って消さないため）

### 時計パネル
- パネル幅 340px を境に「狭い／広い」でアナログ・デジタルの表示を個別にON/OFF可
- デザイン各3種: アナログ（クラシック／ミニマル／ナンバー）、デジタル（スタンダード／LED／モダン）
- LEDはテーマ連動: ライト=LCD液晶風、ダーク=緑発光LED風
- 横画面ではアナログとデジタルを縦並び、縦画面では横並びに自動切替

### カレンダーパネル
- デフォルトカレンダーの**当日の予定**を表示（終日→時刻順、場所付き）

### レイアウト・テーマ
- 時計／ToDo／予定の3パネル制。**表示・非表示と並び順**を設定画面（⚙）から変更可
- 画面の縦横を自動判別（横画面=左右分割、縦画面=上下分割）
- 時計・予定パネルは**スプリッタのドラッグ**でサイズ調整。設定はサーバー側に保存され再起動後も維持
- テーマ: ライト／ダーク／自動（18時〜6時はダーク）

### 運用向け機能
- 5分ごとの自動更新＋右上の⟳ボタンで手動更新（通信中はボタンが回転）
- 更新失敗時は前回データを表示し続け、**原因別の対処法**（ネットワーク確認／再読み込み・再ログイン／PC再起動）をバーに表示。1分ごとに自動再試行
- ヘッダー右上に最終更新日時を表示
- 📱ボタン: Google Tasks を開くQRコード（スマホからのタスク追加・編集用）

## セットアップ

1. [script.google.com](https://script.google.com) で新規プロジェクトを作成し、
   `Code.gs` と `index.html`（ファイル名はこの通りに）を貼り付けて保存
2. 左メニュー「サービス」から拡張サービスを追加
   - **Tasks API**（識別子: `Tasks`）
   - **People API**（識別子: `People`）
   - **Google Chat API**（識別子: `Chat`）… スペース名表示に使用
   - ※カレンダーは標準サービスのため追加不要
3. 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **自分のみ**
4. 発行されたURL（`/exec`）に初回アクセスし、権限を承認
   （タスク・連絡先・Chat・カレンダー・Gmail送信）
5. サイネージPCで対象のGoogleアカウントにログインし、Chromeをキオスクモードで起動
   ```
   chrome --kiosk https://script.google.com/macros/s/xxxxx/exec
   ```

### コード更新時の反映方法

「デプロイ」→「**デプロイを管理**」→ 鉛筆アイコン →「新バージョン」→「デプロイ」。
この方法ならURLが変わらず、キオスク起動設定はそのままで済みます
（「新しいデプロイ」を作ると別URLになるので注意）。
開発中は「デプロイをテスト」の `/dev` URLを使うと保存だけで最新コードが反映されます。

### 点灯スケジュール（OS側で設定）

- シャットダウン: タスクスケジューラで毎日（または平日のみ）`shutdown /s /t 0`
- 起動: BIOS/UEFI の RTCアラーム（Power On By RTC / Auto On Time 等）で電源ON
  - BIOSに曜日指定がない場合は、毎日起動＋土日はログオン時に即シャットダウンするタスクで代替
- 起動時表示: スタートアップにキオスクChromeのショートカットを登録

## 設定（⚙）項目一覧

- テーマ（ライト／ダーク／自動）
- パネルの表示・非表示と並び順（時計／ToDo／予定）
- タスクの並び順（期限順／Google Tasksの順／タイトル順）
- 時計の表示内容（狭い時・広い時 × アナログ・デジタル）
- アナログ／デジタルのデザイン（各3種）

## 既知の制約（Google API仕様によるもの）

- **スペースの他メンバーのタスクは表示できない**
  Tasks API は「自分にアサインされたタスク」のみ返します。スペース全体のタスク一覧を
  取得するAPIは存在しません（アサイン管理は依頼メール機能で代替）。
- **他者へのタスクアサインはAPIから不可**（依頼メール送信で代替）
- **期限は日付のみ**。時刻の設定・表示はできません（Tasks APIの仕様）
- アサイン由来タスクをAPIで削除すると**スペース/Docs側の全員分も削除される**ため、
  本アプリでは削除メニューを非表示にしています
- 連絡先・スペース名は6時間キャッシュされます（変更の反映に最大6時間）
- 依頼メールの「依頼期限」は本文への記載のみで、相手のタスクとして登録はされません

## 動作環境

- 表示: PC上のChrome（キオスクモード推奨）。Fire TV でも Fully Kiosk Browser 等で動作可
  （デプロイのアクセス設定変更が必要）
- アカウント: Google Workspace 推奨（組織ディレクトリからの宛先取得に使用）。
  個人アカウントでも動作します（宛先は個人の連絡先＋その他の連絡先から取得）
